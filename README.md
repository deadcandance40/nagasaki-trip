# Nagasaki Cycle & Trip Planner 🚴🗾

A mobile-first, single-page web app for tracking a cycling trip through Nagasaki, Japan.

![License](https://img.shields.io/badge/license-MIT-blue.svg)
![Made with](https://img.shields.io/badge/made%20with-HTML%2FJS%2FTailwind-orange.svg)

## 🚀 Live Demo

Once deployed to GitHub Pages, access your trip planner at:
```
https://YOUR_USERNAME.github.io/nagasaki-trip
```

## 📱 Features

- **Trip Timeline** — Ferry schedules, hotel check-in, day-by-day itinerary
- **Cycling Routes** — 6 curated routes (sea & mountain) with elevation profiles
- **Restaurant Guide** — Tabelog top-rated spots in Imari & Nagasaki
- **Onsen Finder** — Hot springs with history, facts & Wiki links
- **Nightlife Guide** — Bars, night views, and the Nagasaki "shime" ritual
- **Interactive Map** — Leaflet map with all key locations
- **Check-in System** — Track progress with localStorage persistence
- **Strava Integration** — Ready to connect (placeholder modal)
- **Dark Mode UI** — Glassmorphism design, mobile-optimized

## 🛠️ Tech Stack

- HTML5 + Vanilla JavaScript
- Tailwind CSS (via CDN)
- Leaflet.js (maps)
- LocalStorage (check-ins)
- No build step required

## 📦 Deployment (GitHub Pages)

### Step 1: Create Repository
1. Go to [github.com/new](https://github.com/new)
2. Name: `nagasaki-trip`
3. Make it **Public**
4. Check **"Add a README file"**
5. Click **Create repository**

### Step 2: Upload Files
1. In your new repo, click **"Add file" → "Upload files"**
2. Drag & drop both `index.html` and this `README.md`
3. Click **"Commit changes"**

### Step 3: Enable GitHub Pages
1. Go to **Settings** → **Pages** (left sidebar)
2. Under **Source**, select **Deploy from a branch**
3. Select **main** branch, folder **/(root)**
4. Click **Save**
5. Wait 1-2 minutes for deployment
6. Your URL will appear at the top: `https://YOUR_USERNAME.github.io/nagasaki-trip`

### Step 4: Share with Friend
- Send the GitHub Pages URL via LINE/WhatsApp
- Both of you: open on phone → **"Add to Home Screen"**
- Done! 🎉
